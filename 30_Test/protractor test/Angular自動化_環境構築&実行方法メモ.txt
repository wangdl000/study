■ プラグインのインストール順番
npm cache clean
npm install -g protractor
npm install -g grunt
npm install -g grunt-cli

mkdir C:\opt\workspace\angular
※ 「C:\opt\workspace\angular」は任意的なフォルダ

cd C:\opt\workspace\angular

npm init
npm install grunt --save-dev
npm install -g coffee-script

npm install grunt-protractor-runner --save-dev
npm i screenshot-protractor
npm install xlsx
npm install -g voc


■ webdriver-manager起動方法

※ 「jmas」を自分の端末ユーザーに入り換えてください。
cd C:\Users\jmas\AppData\Roaming\npm

set PROXY=http://proxy.jmas.co.jp:8080
set HTTP_PROXY=%PROXY%
set HTTPS_PROXY=%PROXY% 
webdriver-manager update

webdriver-manager start

■ テスト実施方法
C:\opt\workspace\ProtoractorTest

	・Chrome：
protractor conf.chrome.pc.js
protractor conf.chrome.sp.js

	・Firefox：
protractor conf.firefox.js
		
	・Mac Safari
protractor conf.safari.js
	・Windows10 Edge
protractor conf.edge.js
	・IE
protractor conf.ie.js

■ 説明
	・「_PC」：PC系のテストケース
	・「_SP」：SP系のテストケース
	※ 「_PC」ファイルと「_SP」ファイルに「**ログイン　開始***」というコメントの以下の内容は同じです。共通化しに易しい為に、「_PC」ファイルにSPの判定処理も残っています。
	
	・「_chk」：画面項目入力チェックに対するエラーケース
	・「_ex」：画面処理の異常ケース

	・セッション情報に影響がある為、「multiCapabilities」を使えません。順番でテストケースを実施する

■ IEブラウザの場合
① IEブラウザ設定
「Setting for selenium in ie.png」画像よりIEブラウザを設定する。

② webdriver（IE用）をの更新、起動
cd C:\Users\jmas\AppData\Roaming\npm

set PROXY=http://proxy.jmas.co.jp:8080
set HTTP_PROXY=%PROXY%
set HTTPS_PROXY=%PROXY%
webdriver-manager update --ie

webdriver-manager start

※ 上記②はできない場合、以下の命令でwebdriver（IE用）を起動
java -Dwebdriver.ie.driver=C:\Users\jmas\AppData\Roaming\npm\node_modules\protractor\node_modules\webdriver-manager\selenium\IEDriverServer3.4.0.exe -jar C:\Users\jmas\AppData\Roaming\npm\node_modules\protractor\node_modules\webdriver-manager\selenium\selenium-server-standalone-3.4.0.jar

③ テストケースの実行
protractor conf.ie.js
